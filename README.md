# sprintReview-Retrospective

#How do I interpret user needs and implement them into a program? How does creating “user stories” help with this?
We talk to the stakeholder and disscuss the features they want from there we create them into user stories. This helps by giving devs
a way to understand what the user truly needs. Then the team disscusses what can be done and starts to create the backlog.
#How do I approach developing programs? What Agile processes do I hope to incorporate into my future development work?
To start to approach a program that needs to be done I would plan it out. Read the user stories to understand requirments. Make
estimations as a team to see what can be done. Then get a tester to make tests that need to pass or fail then develop the work. If ever stuck I will ask for clarification. An agile process I will most likey incoperate willl be daily stand ups. The reason being i think this meeting helps the most in keeping the team aligned and keeps obstacles away.
#What does it mean to be a good team member in software development?
To be a good TEAM member we have to communicate with each other this means asking for help, sharing updates, and clarifying anything.
Being able to adapt with what ever changes come and can respond to feedback positivly. Also can work with your team as in if they need help you can help them too.
